# Repo Safety

Use this rule for all work in $repoRelative.

- Preserve existing architecture and local conventions.
- Keep AGENTS.md concise; put deeper workflows in .agent.
- Do not touch secrets, credentials, generated dependency folders, or deployment config without approval.
- Do not run production-impacting commands without approval.
- Prefer focused changes and targeted verification.
- If the task affects user-facing behavior, call out assumptions and verification gaps.
