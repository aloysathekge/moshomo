# Moshomo Scaffold Plan

Use this command before creating or restructuring the Moshomo app workspace.

## Read First

1. `AGENTS.md`
2. `PRD.txt`
3. `.agent/rules/product-boundaries.md`
4. `.agent/rules/monorepo-architecture.md`
5. `.agent/rules/pori-integration.md`
6. `.agent/progress/current.md`

## Output Required Before Edits

- Proposed repo shape.
- Apps and packages to create.
- Source-of-truth documents to add.
- Verification commands for each workspace.
- First vertical slice to build.
- Risks, assumptions, and approval-gated actions.

## Rules

- Do not scaffold the monorepo until the proposed shape is clear.
- Keep V1 focused on employee management, leave, smart shifts, and the Pori assistant.
- Exclude payroll and attendance features unless the user changes scope.
